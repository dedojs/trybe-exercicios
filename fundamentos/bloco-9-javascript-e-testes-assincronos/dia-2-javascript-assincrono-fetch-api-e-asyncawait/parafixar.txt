O que é API?
Application Programming Interface = permite que aplicações se comuniquem umas com as outras servindo de ponte entre elas. 

O que é um código que é executado de modo assíncrono? Qual é a diferença disso para um código que é executado de modo síncrono?
É um código que é executado em segundo plano, e seu retorno acontece quando ela finaliza a sua execução. 
O modo sincrono é exxecutado de forma sequencial, e seu retorno acontece em ordem.

Quando você tem que enfileirar várias callbacks , que problema surge?
Qaundo muitas callbacks são enfileiradas acontece o chamado CALLBACK HELL. 


Por que as Promises são uma forma de resolver esse problema?
As promisses são assincronas, sua execução não trava o fluxo das outras funções.


Quando você vai se comunicar com uma API , tal comunicação deve ser síncrona ou assíncrona? Lembre-se de que o serviço ao qual você está se conectando pode demorar muito ou pouco para dar retorno, pode estar fora do ar etc.
A comunicação é feita sempre de forma assincrona através das promisses.


Dada a resposta anterior, o que é fetch e para que ele serve?
Fetch é um função assincrona , baseada em uma promisse. Ela é utilizada para fazer chmadas as URLS das APis.
